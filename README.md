# Adobe CJA Samples

This repository contains a curated collection of **Adobe Customer Journey Analytics (CJA)** workspace templates, supporting documentation, and visual examples developed as part of ongoing hands-on work in the CJA Sandbox environment.

The goal of this repository is to demonstrate:
- Practical CJA analysis patterns  
- Reusable workspace design templates  
- Clear, stakeholder-focused insight storytelling  
- Cross-channel measurement strategies  

---

## 📁 Repository Structure

Each folder represents a standalone analysis use case and contains:

- **Markdown Template**  
  A reusable template describing the analysis structure, KPIs, and insights framework  

- **PDF Export**  
  A representative CJA Workspace project export  

- **PNG Screenshot**  
  A quick visual reference of the workspace  

---

## 📊 Current Samples

### 1. Data View Summary
📁 `Data View Summary`

**Purpose:**  
Provides a structured overview of a CJA Data View, including key metrics, dimensions, and usage guidance.

**Contents:**
- Markdown template for documenting a Data View [/Data View Summary/CJA Sandbox Template - Data View Summary.md]
- Workspace-style summary approach
- Supporting PDF[/Data View Summary/Data View Summary.pdf] and screenshot[/Data View Summary/Data View Summary.png]

### 2. Digital to Offline Product Conversion
📁 `Digital to Offline Product Conversion`

**Purpose:**  
Analyzes how digital interactions influence offline (POS / call center) purchases.

**Key Concepts:**
- Cross-channel journey stitching  
- Digital → Offline attribution patterns  
- Event-based vs session-based analysis  
- Identification of non-digital conversion paths  

**Contents:**
- Template for cross-channel analysis
- Example workspace output
- Supporting visuals

## 🧠 How to Use This Repository

These templates are designed to be:

- **Reusable** → Adapt for new CJA projects  
- **Instructional** → Learn how to structure analysis panels  
- **Portfolio-ready** → Demonstrate real-world analytics thinking  
- **Stakeholder-friendly** → Emphasize insights over raw data  

You can:
- Copy Markdown templates into your own documentation workflow (e.g., Obsidian)
- Recreate the panels in your own CJA environment
- Use the structure for stakeholder presentations
